# 📊 MooldangBot 백엔드 기능 분석 및 미구현 UI 리스트업

본 문서는 현재 **MooldangBot** 및 **MooldangAPI** 백엔드에 구현된 기능을 분석하고, 이와 매핑되는 프론트엔드(Studio, Admin, Overlay)의 구현 현황을 비교하여 향후 개발이 필요한 UI 리스트를 정리합니다.

---

## 1. 백엔드 기능 분석 (도메인별 구현 현황)

백엔드는 최근 아키텍처 단순화 작업을 통해 매우 견고하고 모듈화된 구조를 갖추고 있으며, 대부분의 핵심 비즈니스 로직이 API 엔드포인트로 노출되어 있습니다.

| 도메인 | 주요 기능 | 구현 상태 | 비고 |
| :--- | :--- | :--- | :--- |
| **치지직 연동** | 채널/방송 정보 조회, 실시간 채팅 엔진, 인증/세션 관리 | 🟢 완성 | `MooldangBot.ChzzkAPI` 사용 |
| **명령어 관리** | 사용자 정의 명령어 추가/수정/삭제, 실행 로그 | 🟢 완성 | `MooldangBot.Modules.Commands` |
| **노래 신청 시스템** | 곡 신청 큐, 라이브러리 관리, 신청 설정(금액, 권한) | 🟢 완성 | `MooldangBot.Modules.SongBook` |
| **룰렛 시스템** | 룰렛 생성, 베팅 처리, 결과 산출 | 🟢 완성 | `MooldangBot.Modules.Roulette` |
| **포인트/경제** | 채팅 포인트 획득, 포인트 사용/선물, 원장(Ledger) 관리 | 🟢 완성 | `MooldangBot.Modules.Point` |
| **아바타 설정** | 방송용 아바타 프리셋 및 파라미터 관리 | 🟢 완성 | `AvatarSettingsController` |
| **오버레이 관리** | 오버레이 프리셋 저장, 레이아웃 구성 저장 | 🟢 완성 | `MasterOverlayController` |
| **주기적 메시지** | 특정 간격 자동 메시지 송출 | 🟢 핵심 로직 완료 | `PeriodicMessageController` |
| **시스템 모니터링** | 봇 상태, 하트비트, 카오스 테스트 | 🟢 완성 | `AdminStatusController` 등 |

## 3. 미구현 UI 및 위젯 리스트업 (Priority 순)

백엔드에는 이미 로직이 존재하지만, 사용자가 시차 없이 관리하거나 방송에 노출할 수 없는 요소들입니다.

### 🚀 Priority 1: 스트리머 필수 관리 도구 (Studio)
*   **주기적 메시지(Periodic Message) 설정 페이지**: 자동 홍보/공지 메시지를 UI에서 쉽게 예약하고 관리하는 화면.
*   **룰렛(Roulette) 관리 페이지**: 현재 진행 가능한 룰렛 리스트와 당첨 확률, 베팅 설정을 관리하는 화면.
*   **곡 라이브러리(Song Library) 통합 관리**: 스트리머의 고유 곡목 리스트를 관리하고, 일괄 추가/삭제하는 기능.
*   **포인트(Point) 설정 및 로그**: 시청자별 포인트 현황 조회 및 획득 정책(배수 등) 설정 UI.

### 📺 Priority 2: 방송용 필수 위젯 (Overlay)
*   **노래 신청(Song Queue) 표시 위젯**: 현재 대기 중인 신청곡 리스트를 실시간으로 방송 화면에 노출.
*   **Now Playing 위젯**: 현재 재생 중인 곡의 정보를 시운전 UI로 제공하는 컴포넌트.
*   **룰렛 애니메이션 위젯**: 룰렛 결과 산출 시 시각적 효과(회전 등) 제공.
*   **채팅 창(Chat Box) 커스텀 오버레이**: 치지직 채팅을 수신하여 방송 화면에 예쁘게 송출.

### 🎭 Priority 3: 개인화 및 고급 설정 (Studio/Admin)
*   **아바타(Avatar) 설정 화면**: 방송용 아바타의 세부 파라미터(눈, 입 등)를 실시간 프리뷰하며 조절.
*   **오버레이 배치 메이커 (WYSIWYG)**: 드래그 앤 드롭으로 위젯 위치를 잡고 프리셋으로 저장하는 화면.
*   **Admin 전체 모니터링**: 봇의 전역 상태, 사용량, 포인트 통계 등 시스템 관리 환경.

현재 백엔드 인프라가 매우 견고하므로, 이제는 **"시청자 경험(Overlay)"**과 **"스트리머 편의성(Studio)"**을 강화할 때입니다.

1.  **단기 (Next Step)**: 룰렛 관리(Studio) 및 룰렛 애니메이션(Overlay) 구현. (시청자 참여 유도)
2.  **중기**: 오버레이 위젯군(노래신청 큐, Now Playing) 확장 및 오버레이 관리자 UI 구축.
3.  **장기**: 아바타 미세 조정, AI(Iamf) 연동 대시보드 및 전체 관리자(Admin) 기능 완성.
