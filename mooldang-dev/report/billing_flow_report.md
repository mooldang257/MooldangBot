# 명령어 비용 차감 로직 순서도

명령어 실행 시 발생하는 선결제(Pre-billing) 및 후속 차감 로직의 흐름을 나타낸 텍스트 기반 순서도입니다. 이번에 수정된 `!신청` 명령어의 동적 비용 평가(Dynamic Cost) 로직도 함께 표현되어 있습니다.

```text
[ 📩 채팅 메시지 수신 ]
       │
       ▼
[ 🔍 명령어 매칭 검사 ] ──(명령어 아님)──▶ [ 🧀 후원금(치즈) 확인 ] ──(없음)──▶ (종료)
       │                                       │
       ▼ (명령어 매칭됨)                       ▼ (있음)
[ 🛡️ 선결제 대상 필터링 ]                [ 🪙 적립 전용 가상 명령어 ]
  (※ 이번 패치 부분!                           │
   SongRequest 기능은 0원 처리)                │
       │                                       │
       ▼                                       │
[ 🧮 총 비용(totalCost) 계산 ] ◀───────────────┘
       │
       ▼
[ 💸 공통 선결제(Pre-billing) 진입 ]
       │
       ├─▶ (결제할 비용=0 & 후원금=0) ────▶ [ ✅ 결제 패스 ] ─────────┐
       │                                                              │
       ▼ (비용>0 또는 후원금>0)                                       │
[ 💰 지갑 적립 처리 ]                                                 │
  (채팅에 실린 치즈 후원금이 있다면 지갑에 먼저 적립)                 │
       │                                                              │
       ▼                                                              │
[ 💳 재화 타입 확인 및 지갑 잔액 차감 ]                               │
       │                                                              │
       ├─▶ (일반 포인트) ──▶ 잔액 검사 ──(부족)──▶ [ ❌ 실패 알림 ] │
       │                         │                                    │
       │                         └─(충분)──▶ 포인트 차감 ────────────▶┤
       │                                                              │
       └─▶ (치즈 결제) ────▶ 잔액 검사 ──(부족)──▶ [ ❌ 실패 알림 ] │
                                 │                                    │
                                 └─(충분)──▶ 치즈 잔액 차감 ─────────▶┤
                                                                      │
       ┌──────────────────────────────────────────────────────────────┘
       │
       ▼
[ 📢 명령어 실행 이벤트 발행 ] (여기부터 각 기능별 개별 로직 실행)
       │
       ▼
[ 🎵 곡 신청(SongRequest) 비동기 로직 시작 ]
       │
       ▼
[ 📖 노래책 검색 및 실제 단가(requiredPoints) 확인 ]
  (예: 역광 = 0치즈, 미등록곡 = 1000치즈)
       │
       ├─▶ (0 치즈 무료 곡) ────▶ [ 🟢 무료 신청곡 등록 완료! ] (종료)
       │
       ▼ (비용이 0보다 큼)
[ 💵 곡별 후원금 누적 검사 ]
  (기존에 누적된 치즈 + 방금 채팅에 실린 치즈 후원금 합산)
       │
       ├─▶ (단가 이상 모임) ────▶ [ 🟢 신청곡 등록 및 누적치즈 초기화! ] (종료)
       │
       ▼ (단가 부족)
[ ⏳ 누적 처리 및 부족 알림 ] 
  "신청 비용이 부족합니다. (현재 500/1000 치즈 누적 중)" (종료)
```

## 핵심 요약
1. **상단 (공통 선결제)**: 이번 패치로 인해 `!신청` 명령어는 상단에서 비용이 0원으로 계산되어, 지갑 잔액이 없어도 무사히 **결제 패스**를 통과합니다.
2. **하단 (개별 로직)**: 무사히 하단 로직에 진입한 뒤, 실제 곡이 무료(0치즈)인지 유료(1000치즈 등)인지 확인하여 각 단가에 맞게 곡을 등록하거나 후원금을 누적하게 됩니다.
