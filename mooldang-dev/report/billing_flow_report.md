# 명령어 및 후원금 결제 로직 순서도 (정책 B: 지갑 연동 기반)

최신 결제 아키텍처(지갑 전액 선적립 및 단일 원자적 결제, 크라우드 펀딩 누적 제거)의 흐름을 나타낸 텍스트 기반 순서도입니다.

```text
[ 📩 채팅 메시지 수신 ]
       │
       ▼
[ 🔍 명령어 매칭 검사 ] ──(명령어 아님)──▶ [ 🧀 후원금(치즈) 확인 ] ──(없음)──▶ (종료)
       │                                       │
       ▼ (명령어 매칭됨)                       ▼ (있음)
[ 🛡️ 동적 비용 명령어 확인 ]             [ 🪙 적립 전용 가상 명령어 ]
  (IsDynamicCost 속성 검사)                    │
       │                                       │
       ├─▶ (동적 비용 O) ──▶ TotalCost = 0     │
       └─▶ (동적 비용 X) ──▶ TotalCost 계산    │
       │                                       │
       ▼                                       │
[ 💸 공통 선결제(Pre-billing) ] ◀──────────────┘
       │
       ├─▶ (비용=0 & 후원금=0) ────▶ [ ✅ 결제 패스 ] ─────────┐
       │                                                       │
       ▼ (비용>0 또는 후원금>0)                                │
[ 💰 지갑 전액 적립 (DonationPoint DB Update) ]                │
  (채팅에 실린 실시간 치즈 후원금 무조건 100% 적립)            │
       │                                                       │
       ▼                                                       │
[ 💳 재화 타입 확인 및 공통 선결제 잔액 차감 ]                 │
  (TotalCost 만큼 지갑/포인트에서 차감. 단, 동적비용은 0원)    │
       │                                                       │
       ├─▶ (잔액 부족) ──▶ [ ❌ 실패 알림 ]                    │
       └─▶ (충분) ───────▶ [ ✅ 결제 성공 처리 ] ──────────────┤
                                                               │
       ┌───────────────────────────────────────────────────────┘
       ▼
[ 📢 CommandExecutedEvent 발행 (비동기) ]
       │
       ▼
[ 🎵 개별 로직: AddSongRequestCommand 진입 ]
       │
       ▼
[ 📖 노래책 검색 및 요구 단가(requiredPoints) 확인 ]
       │
       ├─▶ (0 치즈 무료 곡) ────▶ [ 🟢 무료 신청곡 등록 완료! ] (종료)
       │
       ▼ (요구 단가 > 0)
[ ⚖️ 단일 지갑 원자적 결제 (MariaDB Transaction) ]
  (DeductDonationPointsCommand 호출. DB의 지갑 잔액으로만 일시불 결제)
       │
       ├─▶ (지갑 잔액 < 단가) ──▶ [ ❌ 비용 부족 알림 ] (종료)
       │                          (※ 실시간 후원금은 윗 단계에서 지갑에 이미 보존됨)
       │
       ▼ (지갑 잔액 >= 단가)
[ 🟢 지갑에서 단가만큼 차감 및 곡 큐(Queue) 등록 시도 ]
       │
       ├─▶ (DB 곡 등록 성공) ──────▶ [ 🎉 곡 등록 완료 및 알림! ] (종료)
       │
       ▼ (DB 곡 등록 실패 - 장애, 중복 등)
[ 🚨 보상 트랜잭션 (Refund) ] ──▶ [ 💸 차감된 치즈 지갑으로 전액 환불 ] ──▶ [ ❌ 실패 알림 ] (종료)
```

## 💡 핵심 요약
1. **지갑 중심 결제 (Double Dipping 원천 차단)**: 채팅에 실린 치즈 후원금은 무조건 시청자의 지갑(DonationPoint)에 100% 사전 적립된 후, 신청 단가에 따라 지갑 잔액 내에서만 원자적으로 차감됩니다. 돈이 누락되거나 증발하지 않습니다.
2. **보상 트랜잭션 (Compensating Transaction)**: 지갑에서 차감은 성공했으나 예기치 못한 DB 오류로 곡이 정상 등록되지 않았을 경우, `try-catch` 블록을 통해 차감된 금액을 자동으로 환불해 주어 안전성을 극대화했습니다.
3. **크라우드 펀딩 제거**: 여러 명이 조금씩 모아 곡을 띄우는 누적 시스템을 완전히 폐지하여 시스템 설계의 단순성(Simplicity)과 동시성 안전(Concurrency Safety)을 대폭 향상했습니다.
