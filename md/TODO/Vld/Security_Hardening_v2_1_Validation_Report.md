## 🎯 2. 상세 수정 내용 및 의도 (Rethink)

### 2.1 StreamerProfile: API 자격 증명 봉인
- **수정**: `ApiClientId`를 평문에서 암호화 대상으로 전환.
- **의도**: 스트리머별 개별 앱의 자격 증명이 DB에 노출되지 않도록 하여, 혹시 모를 DB 탈취 시에도 API 키 오남용을 원천 차단했습니다.

### 2.2 RouletteLog: 시청자 식별자의 기밀성 확보
- **수정**: UID 필드 추가 및 암호화 적용.
- **의도**: 기존에는 시청자 닉네임만 기록했으나, 정확한 통계와 보안을 위해 고유 UID를 저장합니다. 단, 개인정보이므로 **수호자의 암호(DataProtection)**를 통해 암호화하여 저장합니다.

### 2.3 Search Hash 전략 (ViewerUidHash)
- **수정**: 원본 UID와 별도로 SHA-256 기반의 `ViewerUidHash` 필드 생성.
- **의도**: 암호화된 `ViewerUid` 필드는 SQL `WHERE` 절에서 검색이 불가능합니다. 이를 해결하기 위해 원본을 알 수 없는 해시값을 별도로 저장하고 인덱스를 걸어, **보안을 유지하면서도 검색 성능을 극대화**했습니다.

### 2.4 Infrastructure: 회복 탄력성 강화
- **수정**: 마이그레이션 도구에 `EnableRetryOnFailure()` 적용.
- **의도**: 룰렛 로그처럼 데이터가 방대한 테이블의 스키마 변경 시, 네트워크 지연이나 DB 부하로 인해 연결이 잠시 끊기더라도 시스템이 포기하지 않고 재시도하여 **마이그레이션 무결성**을 지켜내도록 설계했습니다.
