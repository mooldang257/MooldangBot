# [Project Osiris]: 함선 운영 지능 가이드 (Operational Intelligence Guide)

본 문서는 오시리스 함선의 데이터를 관리하고 시스템 상태를 관제하는 **선장(스트리머)용 운영 가이드**입니다. 특히 Phase 16~18을 거치며 완성된 **'심연의 시련(Resilience)'** 및 **'이지스의 기록관(Observability)'** 체계를 포함합니다.

---

## 1. 📂 데이터 관리 및 장부 (Data & Ledger)

### 💰 천상의 장부 (Celestial Ledger)
- **24시간 자동 숙청**: 상세 거래 로그(`log_point_transactions`)는 매일 한 번씩 **최근 30일**치만 남기고 자동으로 정리됩니다.
- **영점 조절 (Zeroing Worker)**: 분산 환경에서의 카운트 누수를 방지하기 위해 24시간마다 Redis의 실시간 카운트를 DB 데이터와 동기화하여 교정(Drift Correction)합니다.

### 💬 채팅 및 🎰 룰렛 로그 관리
- **공통 보관 기간**: 채팅 로그(`log_chat_interactions`)와 룰렛 로그(`roulette_logs`)는 모두 **최근 90일**간 보관됩니다. (매일 1회 정리)
- **지능형 데이터 보호**: 룰렛 로그 정리 시, **스트리머가 리워드를 아직 지급하지 않은 미수행(`Pending`) 기록은 90일이 지나더라도 절대 삭제되지 않도록 보호**합니다.

## 2. 🚨 지능형 조기 경보 (Aegis Alarm)

[Phase 15] 함대가 여러 대의 인스턴스로 운용됨에 따라 중복 알림을 방지하고 정확한 상태를 전달하는 지능형 경보 체계입니다.

### 🧱 알림 마스터 (Alert Master via RedLock)
- 함대 내 모든 인스턴스가 경보를 점검하지만, **RedLock**으로 선출된 단 한 대의 '알림 마스터'만 실제 알림을 발송하여 중복을 차단합니다.

### 🛡️ 안티 플래핑 (Anti-Flapping via Redis)
- 동일한 사유의 장애 알림은 **1시간 동안 1회**만 발송됩니다. Redis TTL을 활용하여 장애 복구 중 발생하는 일시적인 흔들림(Flapping) 알림을 필터링합니다.

### 🖥️ 감시 대상 및 임계치
- **메모리 과식**: 인스턴스당 메모리 점유율이 **1GB(1024MB)**를 초과할 경우 경보.
- **함대 이탈**: 5분 이상 맥박(Pulse)이 끊긴 인스턴스 발견 시 즉각 알림.

## 3. 📈 실시간 함교 관제 (Fleet Monitoring)

Admin 대시보드를 통해 함대 전체의 생존 신호를 실시간으로 확인할 수 있습니다.

### 🖥️ 시스템 펄스 (Fleet ECG)
- **위치**: `/admin/system-pulse`
- **내용**: 각 인스턴스(Machine)별 메모리 사용량, CPU 시간, 백그라운드 워커들의 생존 여부를 통합 시각화합니다.
- **맥박(Heartbeat)**: 인스턴스별 상태뿐만 아니라, 모든 백그라운드 워커가 `BaseHybridWorker` 엔진을 통해 자신의 생존 신호를 자동으로 보고합니다. 이를 통해 어떤 워커가 멈췄는지 대시보드에서 즉시 확인 가능합니다.

### 🛖 함대의 통합 기억 (Centralized Logging)
- **Grafana Loki**: 모든 컨테이너의 로그가 한곳으로 모입니다. 특정 시점의 장애를 추적할 때 인스턴스 ID와 TraceID를 사용하여 '범인'을 단번에 식별할 수 있습니다.
- **TraceID 추적**: 하나의 요청이 여러 서비스를 거쳐도 동일한 TraceID가 유지되므로, 전체 흐름을 전수 조사할 수 있습니다.

## 4. 🌊 심연의 시련 (Abyssal Trials)

[Phase 18] 함선이 극한의 환경에서도 생존할 수 있는지 테스트하고, 어떤 상황에서도 데이터를 사수하는 매커니즘입니다.

### 🧪 카오스 시뮬레이션 (Simulated Faults)
선장님은 Admin 함교나 API를 통해 인위적인 시련을 주입할 수 있습니다.
- **Redis Panic**: Redis가 중단된 상황을 가정하여, 시스템이 로컬 폴백 모드로 잘 전환되는지 테스트합니다.
- **API Delay**: 네트워크 지연이 발생했을 때 함선이 비명을 지르지 않고 유연하게 대응하는지 확인합니다.

### 💾 익산 보험 (Iksan Insurance)
이사는 선장님에게도 오시리스에게도 큰 시련입니다.
- **파일 덤프**: 서버 종료 시 미처리된 포인트 데이터는 즉시 `temp_point_queue.json` 파일로 안전하게 대피합니다.
- **자동 복구**: 서버를 다시 켜는 순간, 오시리스는 대피시켰던 데이터를 가장 먼저 읽어 들여 DB에 안전하게 적재합니다.

## ⚙️ 5. 필수 환경 변수 (.env)

운영 지능 가동을 위해 아래 변수의 정확한 설정이 필요합니다.

```bash
# 디스코드 웹훅 (이지스의 파수꾼)
DISCORD_CRITICAL_URL="https://discord.com/api/webhooks/..." # 긴급 장애 및 과식 알림
DISCORD_STATUS_URL="https://discord.com/api/webhooks/..."   # 주간 리포트 및 일반 상태 보고

# [v18.0] 심연의 시련 활성화
ENABLE_CHAOS_TRIALS=false  # 실제 방송 중에는 false 권장
```
