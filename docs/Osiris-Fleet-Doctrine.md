# 🔱 오시리스 함대 교리 (Osiris Fleet Doctrine: Salvo V2)

본 문서는 오시리스 함대의 모든 개발 대원이 준수해야 할 불변의 아키텍처 원칙과 코딩 교리를 선언합니다. 우리 함선은 고도의 분산 지능과 자율 복구 능력을 지향하며, 본 교리는 그 질서를 유지하는 근간이 됩니다.

---

## 🏛️ 제1장: EDMH (Event-Driven Modular Hub)
### [자율 반응의 원칙: Choreography over Orchestration]

지휘관(Hub)은 사소한 일에 일일이 간섭하지 않습니다. 사건을 공표할 뿐, 행동은 각 부서가 결정합니다.

1.  **중앙 집권 타파**: `UnifiedCommandHandler`는 이제 직접 명령하지 않습니다. 오직 '사건의 발생'(`CommandExecutedEvent`)을 함대 전역에 통보합니다.
2.  **자율적 반응**: 룰렛, 포인트, 노래책 등의 모듈은 해당 사건을 경청(Listen)하다가, 자신의 임무라 판단되면 독립적으로 행동합니다.
3.  **결합성 제로**: 모듈을 추가하거나 제거할 때 허브의 코드를 수정할 필요가 없어야 합니다.

---

## 聖 제2장: Contracts (Holy Land)
### [계약의 성소: 거룩한 격리]

프로젝트 간의 직접 참조는 금기시됩니다. 모든 소통은 성지(`Contracts`)를 통해서만 이루어집니다.

1.  **인터페이스 기반 소통**: 모듈 간의 함수 호출은 반드시 `IContract` 인터페이스를 거칩니다.
2.  **DTO의 규격화**: 데이터를 주고받을 때는 도메인 엔티티가 아닌, 정제된 DTO 및 Event 모델만을 사용합니다.
3.  **격리의 이점**: 성소만 일치한다면, 함선의 일부가 물리적으로 분리되어 다른 서버(Process)로 이동해도 시스템은 중단 없이 맥동합니다.

---

## 📡 제3장: Integration Messages
### [외교 격리: 함선 기밀 보호]

내부의 언어(`Domain Event`)와 외부의 언어(`Integration Event`)를 분리하여 함선의 기밀과 비즈니스 로직을 보호합니다.

1.  **정보의 선별**: 외부 광통신망(RabbitMQ)으로 메시지를 쏠 때는 도메인의 날것을 보내지 않습니다. 외교 규격(`Integration Message`)으로 가공하여 전파합니다.
2.  **보안 및 유연성**: 내부 데이터 구조가 바뀌어도 외부 통신 규격만 유지된다면 함대 전체의 전열은 흐트러지지 않습니다.

---

## 💖 제4장: Self-Healing Saga
### [자가 치유의 의무: 불사의 신뢰성]

분산 환경에서의 실패를 숙명으로 받아들이고, 이를 스스로 극복하는 지능을 갖춥니다.

1.  **보상 트랜잭션**: "포인트는 깎였는데 룰렛이 실패했다"면, 시스템은 즉시 이를 감지하고 지휘관의 명령 없이도 포인트 자동 환불(`RefundCurrency`)을 집행합니다.
2.  **Saga 감시체계**: `MassTransit Saga State Machine`은 모든 유료 작전의 시작과 끝을 감시하며, 위기 상황 발생 시 20초 이내에 자율 대응합니다.

---

## 🧬 제5장: Genesis Tracing (Correlation ID)
### [혈통 추적의 법칙: 인과의 고리]

함선 전역을 흐르는 모든 데이터는 자신의 조상과 목적을 증명해야 합니다.

1.  **상관관계 ID 전파**: 시청자의 채팅 한 마디가 시발점이 될 때 생성된 `CorrelationId`는 RabbitMQ와 로그, 응답 메시지 끝까지 유지되어야 합니다.
2.  **전황 파악**: 이 ID 하나만으로 Grafana에서 해당 요청의 전 생애 주기를 단 1ms의 오차 없이 역추적할 수 있어야 합니다.
